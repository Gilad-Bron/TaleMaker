Data Model — v2

MongoDB collections and key fields

Collection

User

{
  _id, email, passwordHash,
  username, avatarUrl,
  roles: ["author","player"]
}
Collection

Character

{
  _id, ownerId (User | null),
  name, portrait, backstory,
  stats: { str, int, cha, ... },
  inventory: [{ itemId, qty }],
  health: { current, max },
  resources: { gold, mana },
  isGlobalPremade: bool,
  taleId (null = global premade)
}
Collection — NEW
Added

Enemy

{
  _id, authorId,
  name, portrait, description,
  stats: {
    hp, str, def, spd, ...
  },
  abilities: [{
    name, description,
    effect, cooldown
  }],
  drops: [{
    itemId, chance, qty
  }],
  createdAt, updatedAt
}

Authors build a reusable enemy roster per account.

Combat interactions reference an enemy by enemyId...

...but can override any field inline for that specific encounter.

Drops feed into the player's inventory on victory.

Collection

Tale — full content stored as document

{
  _id, authorId, title, description,
  visibility: "public" | "private" | "invite",
  allowExternalCharacters: bool,
  invitedUserIds: [],
  premadeCharacterIds: [],
  publishedAt,
  chapters: [{
    _id, title, episodes: [{
      _id, title, interactions: [{
        _id, type: "dialog"|"trade"|"combat"|"area", name,
        ...typeSpecificFields
      }]
    }]
  }]
}
Interaction Types (embedded in Tale)

dialog / trade / combat / area

dialog: {
  dialogs: [{ text, options: [{
    label, targetDialogId,
    skillCheck, effects
  }]}]
}
trade: {
  npcName, offers: [Item],
  wants: [Item], conditions
}
combat: {
  enemyId,              ← ref to Enemy
  statOverrides: {...}, ← optional inline
  dropOverrides: [...], ← optional inline
  outcomes: { win: { nextNodeId, effects },
              lose: { nextNodeId, effects } }
}
area: {
  description,
  objects: [{ name, desc,
    interactionId }]
}
Collection

PlaySession

{
  _id, userId, taleId,
  characterSnapshot: { ...Character },
  currentNodeId,
  status: "active"|"complete",
  saveStates: [{
    label, characterSnapshot,
    currentNodeId, savedAt
  }],
  startedAt, completedAt
}

Key Relationships

User → owns → Characters • User → authors → Tales + Enemies • Combat → refs → Enemy (+ inline overrides) • User → plays → PlaySessions • Character snapshot stored in session
