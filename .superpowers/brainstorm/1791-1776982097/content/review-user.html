Entity 1 of 8 — User

The person using the platform. Can be an author, a player, or both.

{
  id:        string
  email:     string
  name:      string
  roles:     "author" | "player" | "both"
  createdAt: Date
}
Notes

— roles determines what the user can do: authors see the workspace, players see the play view
— a user with both can switch between author and player modes
— no avatar or profile fields yet — minimal to start
