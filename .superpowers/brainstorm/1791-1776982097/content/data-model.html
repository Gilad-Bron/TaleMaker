Data Model

MongoDB collections and key fields

Collection

User

{
  _id, email, passwordHash,
  username, avatarUrl,
  roles: ["author","player"]
}
Collection

Character

{
  _id, ownerId (User | null),
  name, portrait, backstory,
  stats: { str, int, cha, ... },
  inventory: [{ itemId, qty }],
  health: { current, max },
  resources: { gold, mana },
  isGlobalPremade: bool,
  taleId (null = global premade)
}
Collection

Tale — full content stored as document

{
  _id, authorId, title, description,
  visibility: "public" | "private" | "invite",
  allowExternalCharacters: bool,
  invitedUserIds: [],
  premadeCharacterIds: [],
  publishedAt,
  chapters: [{
    _id, title, episodes: [{
      _id, title, interactions: [{
        _id, type: "dialog"|"trade"|"combat"|"area",
        name,
        ...typeSpecificFields   ← see Interaction Types below
      }]
    }]
  }]
}
Interaction Types (embedded)

dialog / trade / combat / area

dialog: {
  dialogs: [{ text, options: [{
    label, targetDialogId,
    skillCheck, effects
  }]}]
}
trade: {
  npcName, offers: [Item],
  wants: [Item], conditions
}
combat: {
  enemy: { name, hp, stats },
  rounds, outcomes: { win, lose }
}
area: {
  description,
  objects: [{ name, desc,
    interactionId }]
}
Collection

PlaySession

{
  _id, userId, taleId,
  characterSnapshot: { ...Character },
  currentNodeId,
  status: "active"|"complete",
  saveStates: [{
    label, characterSnapshot,
    currentNodeId, savedAt
  }],
  startedAt, completedAt
}

Key Relationships

User → owns → Characters • User → authors → Tales • User → plays → PlaySessions • Tale embeds full content tree • Character snapshot stored in session
